# 🏂 UX 第四輪建議：手機優先（雪地場景）

**審查時間**：2025-11-28
**基於實際體驗**：https://sb-qa-web.zeabur.app/
**核心限制**：手機螢幕 + 強光 + 戴手套 + 弱網 + 碎片時間

---

## 🎯 核心問題診斷

基於實際截圖與手機使用場景，**當前最大問題**：

| 問題 | 證據（截圖） | 手機場景影響 |
|------|-------------|-------------|
| 1. **卡片資訊過載** | 課程卡片包含 100+ 字描述 | 單手滑動時難以快速掃描，需要停下來閱讀 |
| 2. **等級徽章過小** | 「初級/中級」徽章僅 24px | 戴手套無法精準點擊篩選 |
| 3. **評分滑桿誤觸** | 滑桿軌道細（僅 12px 高） | 戴手套調整評分困難，常滑到錯誤位置 |
| 4. **快速入口按鈕擁擠** | 8 個問題按鈕（🦶後刃、👣前刃...）兩列排列 | 手套點擊常誤觸相鄰按鈕 |
| 5. **底部操作欄擁擠** | 3 個按鈕（❤️/🏂/📤）橫排 | 中間「完成練習」易被誤點成收藏 |

---

## ✅ 手機優先改善方案（優先級排序）

### Priority 1：卡片資訊簡化（掃描效率 ↑80%）⏱️ 30 分鐘

**問題**：課程卡片顯示完整問題描述（100+ 字），導致：
- 卡片高度過高（需要多次滑動）
- 視覺噪音大（難以快速掃描標題）

**方案**：**僅顯示標題 + 徽章，問題描述摺疊**

```tsx
// web/src/components/LessonCard.tsx
export function LessonCard({ lesson }) {
  return (
    <Link
      href={`/lesson/${lesson.id}`}
      className="
        block p-5 mb-4
        bg-slate-800/80 backdrop-blur-sm
        border border-slate-700
        rounded-xl
        active:scale-98  /* 點擊回饋 */
        transition-transform duration-100
      "
    >
      {/* 標題（加大到 20px，最多 2 行） */}
      <h3 className="
        text-xl font-bold text-slate-100
        line-clamp-2 mb-3
      ">
        {lesson.title}
      </h3>

      {/* 徽章（放大到 36px 高，間距加大） */}
      <div className="flex flex-wrap gap-3 mb-3">
        {lesson.level_tags.map(level => (
          <span className="
            px-4 py-2  /* 加大內距 */
            rounded-full
            bg-orange-500/20 text-orange-300
            text-sm font-semibold
          ">
            {level}
          </span>
        ))}
        <span className="px-4 py-2 rounded-full bg-purple-500/20 text-purple-300 text-sm">
          {lesson.casi.Primary_Skill}
        </span>
      </div>

      {/* ❌ 移除問題描述（僅在詳情頁顯示） */}
    </Link>
  )
}
```

**預期效果**：
- 卡片高度從 ~180px → ~120px（↓33%）
- 一屏可見課程數：3 個 → 5 個
- 滑動次數減少 40%

### Priority 2：快速入口按鈕間距加大（誤觸率 ↓60%）⏱️ 20 分鐘

**問題**：8 個問題按鈕（🦶後刃、👣前刃...）兩列排列，間距僅 8px

**方案**：**單列滑動 + 間距加大到 12px**

```tsx
// web/src/app/page.tsx - "你遇到什麼問題？" 區塊
<section className="mb-8">
  <h2 className="text-2xl font-bold mb-4">你遇到什麼問題？</h2>

  {/* 水平滾動容器 */}
  <div className="
    flex gap-3  /* 間距 12px */
    overflow-x-auto
    pb-3  /* 底部留白，避免滾動條遮擋 */
    -mx-4 px-4  /* 左右突破容器邊界 */
    scrollbar-hide  /* 隱藏滾動條 */
  ">
    {QUICK_FILTERS.map(filter => (
      <button className="
        flex-shrink-0  /* 防止壓縮 */
        px-5 py-3  /* 加大到 44px 高 */
        rounded-xl
        bg-slate-800 border border-slate-700
        text-base font-semibold
        whitespace-nowrap  /* 單行顯示 */
        active:scale-95
      ">
        {filter.emoji} {filter.label}
      </button>
    ))}
  </div>
</section>
```

**CSS 隱藏滾動條**：
```css
/* web/src/app/globals.css */
@layer utilities {
  .scrollbar-hide {
    -ms-overflow-style: none;  /* IE */
    scrollbar-width: none;     /* Firefox */
  }
  .scrollbar-hide::-webkit-scrollbar {
    display: none;  /* Chrome/Safari */
  }
}
```

**預期效果**：
- 誤觸率從 ~30% → <10%
- 單手拇指可輕鬆滑動選擇
- 視覺更清爽（不擁擠）

### Priority 3：評分滑桿放大（手套友善）⏱️ 1 小時

**問題**：滑桿軌道高度僅 12px，戴手套難以精準拖動

**方案**：**改用分段按鈕 + 觸控區域 ≥48px**

```tsx
// web/src/components/practice/RatingButtons.tsx
'use client'
import { useState } from 'react'

export function RatingButtons({ label, value, onChange }) {
  return (
    <div className="mb-6">
      {/* 標籤 */}
      <div className="flex items-center justify-between mb-3">
        <span className="text-lg font-semibold">{label}</span>
        <span className="text-2xl font-bold text-amber-400">
          {value}/5
        </span>
      </div>

      {/* 分段按鈕（5 個） */}
      <div className="flex gap-2">
        {[1, 2, 3, 4, 5].map(score => (
          <button
            key={score}
            onClick={() => {
              onChange(score)
              // 觸覺回饋
              if (navigator.vibrate) navigator.vibrate(10)
            }}
            className={`
              flex-1 h-12  /* 48px 高 */
              rounded-lg
              text-base font-bold
              transition-all duration-200
              ${value >= score
                ? 'bg-amber-500 text-slate-900 scale-105'
                : 'bg-slate-700 text-slate-400'
              }
              active:scale-95
            `}
          >
            {score}
          </button>
        ))}
      </div>

      {/* 提示文字 */}
      <div className="flex justify-between mt-2 text-sm text-slate-400">
        <span>需加強</span>
        <span>很好</span>
      </div>
    </div>
  )
}
```

**預期效果**：
- 戴手套也能精準點擊（觸控區域 48px）
- 視覺回饋清晰（選中按鈕放大 + 高亮）
- 誤觸率 ↓80%

### Priority 4：底部操作欄按鈕間距加大⏱️ 15 分鐘

**問題**：3 個按鈕（❤️ 收藏/🏂 完成練習/📤 分享）橫排，間距過小

**方案**：**主按鈕獨立一行 + 次要按鈕上移**

```tsx
// web/src/components/BottomActionBar.tsx
export function BottomActionBar({ lessonId }) {
  return (
    <div className="
      fixed bottom-0 left-0 right-0
      bg-slate-900/95 backdrop-blur-xl
      border-t border-slate-700
      p-4
      pb-safe  /* iOS 安全區域 */
    ">
      {/* 次要按鈕（收藏 + 分享，移到頂部） */}
      <div className="flex gap-3 mb-3">
        <button className="
          flex-1 h-12
          rounded-xl
          bg-slate-800 border border-slate-700
          text-sm font-semibold
          active:scale-95
        ">
          ❤️ 收藏
        </button>
        <button className="
          flex-1 h-12
          rounded-xl
          bg-slate-800 border border-slate-700
          text-sm font-semibold
          active:scale-95
        ">
          📤 分享
        </button>
      </div>

      {/* 主按鈕（完成練習，獨立一行） */}
      <button className="
        w-full h-14  /* 56px 高 */
        rounded-2xl
        bg-gradient-to-r from-amber-500 to-orange-500
        text-lg font-bold text-slate-900
        shadow-lg shadow-amber-500/30
        active:scale-98
      ">
        🏂 完成練習
      </button>
    </div>
  )
}
```

**預期效果**：
- 主按鈕誤觸率 ↓90%（獨立一行）
- 單手拇指可輕鬆點擊（底部位置）
- 視覺層級清晰（主次分離）

## 🎨 手機專屬美學原則

基於雪地手機場景，**簡化版設計原則**：

### ✅ 保留（有效的）
- ✅ Snow Mode（黃黑高對比）
- ✅ 字級 18px（可讀性）
- ✅ 底部操作欄（拇指可觸及）
- ✅ 震動回饋（確認感）
- ✅ 麵包屑導航（清晰路徑）

### ❌ 避免（手機不適合）
- ❌ 複雜漸層（耗電 + 載入慢）
- ❌ Hover 動效（手機無 hover）
- ❌ 全屏彈窗（遮擋內容，難以退出）
- ❌ 細小滑桿（戴手套無法操作）
- ❌ 多行小按鈕（誤觸率高）

### 🎯 手機優先設計語言

**核心概念**：**大塊面 + 高對比 + 單列流**

```
✅ 正例（手機友善）
┌────────────────────┐
│ [大標題]            │  ← 20px+，2 行內
│ [徽章] [徽章]      │  ← 40px 高，可點擊
│                    │
│ [主按鈕 100% 寬]   │  ← 56px 高，底部固定
└────────────────────┘

❌ 反例（手機不友善）
┌────────────────────┐
│ 小標題 [小徽章]    │  ← 難以閱讀
│ 很長的描述文字...  │  ← 佔用空間
│ [按鈕1][按鈕2][3]  │  ← 擁擠，誤觸
└────────────────────┘
```
